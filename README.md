# 计算机机房设备管理系统设计方案 (C语言期末课程设计)

## 一、项目概述

### 1.1 项目名称
计算机机房设备管理系统

### 1.2 项目背景
随着学校信息化建设的不断深入，计算机机房设备数量不断增加，设备管理变得日益复杂。传统的手工记录方式效率低下，容易出错，无法满足现代化管理的需求。因此，开发一个计算机机房设备管理系统，实现设备信息的数字化、规范化管理，具有重要的现实意义。

### 1.3 项目目标
1. 实现机房设备信息的电子化管理
2. 提供设备查询、添加、修改、删除等基本功能
3. 实现设备借用、归还、维修等业务流程管理
4. 生成设备统计报表
5. 提高机房管理效率，降低管理成本

### 1.4 开发环境
- 操作系统：Windows 10/11
- 开发语言：C语言
- 开发工具：Visual Studio Code / Dev-C++
- 编译环境：GCC/MinGW

#### 2.1.3 设备借用管理模块

#### 2.1.4 统计报表模块

### 2.2 非功能需求

## 三、系统设计

### 3.1 系统架构设计
采用模块化设计思想，将系统划分为多个功能模块，各模块之间通过函数调用进行通信。

### 3.2 数据结构设计

#### 3.2.1 设备信息结构体
```c
typedef struct
{
    int id;                 // 设备编号
    char name[50];          // 设备名称
    char type[30];          // 设备类型（台式机，笔记本，CPU，内存等）
    char brand[30];         // 品牌
    char model[30];         // 型号
    Date purchase_date;     // 购买日期
    double price;           // 价格
    char status[20];        // 状态（正常、故障、停用、维修中、报废）
} Device;
```

#### 3.2.2 借用记录结构体
```c
typedef struct
{
    int record_id;                 // 借用记录编号
    int device_id;                 // 设备编号
    Date borrow_date;              // 借用日期
    Date return_date;              // 应归还日期
    Date actual_return_date;       // 实际归还日期
    char status[20];               // 状态（借用中、已归还、逾期）
}Borrow;
```

#### 3.2.3 维修记录结构体
```c
typedef struct {
    int repair_id;          // 维修记录编号
    int device_id;          // 设备编号
    char fault_desc[100];   // 故障描述
    Date repair_date;       // 维修日期
    char repair_person[50]; // 维修人员
    double cost;            // 维修费用
} Repair;
```
#### 3.2.4 日期结构体
```c
typedef struct
{
    int year;
    int month;
    int day;
}Date;
```

### 3.3 文件存储设计
系统采用文件方式存储数据，设计以下数据文件：
1. `device_report.csv` - 设备信息数据文件
2. `borrow_report.csv` - 借用记录数据文件
3. `repair_report.csv` - 维修记录数据文件
